⚠️ Common causes of SRM — investigate these

🤖
Bot or crawler traffic
Automated traffic hitting one variant more than the other. Filter bots before re-running.
⏱️
Delayed experiment trigger
Assignment happens at page load but logging happens after an async event — some users never get logged.
🔁
Redirects in the treatment
If treatment redirects users, some browsers drop the assignment cookie before logging completes.
💾
Cache serving control to treatment users
CDN or browser cache returning the wrong variant. Check cache headers and surrogate keys.
🚪
Incorrect exclusion logic
A filter applied after assignment (e.g. exclude mobile) that affects one variant disproportionately.
